## 三、遗留问题与结论

### 1. 已解决问题

- **设置窗口状态同步问题**
  - 现象：重新打开设置窗口时，偶发看到旧的维度勾选状态。
  - 处理：在 `src/main/tray.ts` 的 `openSettingsFromTray()` 中，打开设置前主动发送一次 `config:updated`。
  - 结果：阶段 5.1 的“重启后配置恢复”链路恢复稳定。

### 2. 仍需人工复核的问题

以下两项不是功能没做，而是自动化环境对透明浮窗中的真实点击不稳定：

1. 展开态自定义 checkbox 的真实鼠标点击
2. 边缘把手的真实鼠标点击恢复

所以最终对外结论应写成：

- **自动化验收：通过**
- **发布前人工复核：必须补做上面 2 项**

## 四、人工验收怎么做（小白版）

下面按顺序做，照着点就行。

### 1. 启动应用

1. 在项目根目录运行：
   ```bash
   npm run dev
   ```
2. 看到桌面上出现一个小浮窗。
3. 右下角系统托盘里能看到应用图标。

**通过标准：**

- 桌面上有浮窗
- 托盘里有图标
- 没报错、没闪退

### 2. 验证空状态

1. 如果还没配厂商，浮窗里应该看到“请配置厂商”。

**通过标准：**

- 不是空白
- 不是崩溃
- 有明确提示你去配置

### 3. 添加智谱

1. 右键托盘图标。
2. 点“设置”。
3. 点“添加厂商”。
4. 选“智谱 CodeGeeX”。
5. 粘贴你的智谱 Token。
6. 点“保存”。

**通过标准：**

- 设置窗口能打开
- 能保存成功
- 浮窗里马上出现智谱这一行

### 4. 验证展开 / 折叠

1. 点击浮窗里智谱这一行。
2. 应该能展开看到“每5小时 Token”和“MCP 每月额度”。
3. 再点一次，应该折叠回一行。

**通过标准：**

- 能展开
- 能折叠
- 展开后能看到多个维度

### 5. 验证勾选维度（这是必须人工补点的一项）

1. 展开浮窗。
2. 先勾上 `MCP 每月额度`。
3. 再取消 `每5小时 Token`。
4. 让浮窗回到折叠态。

**通过标准：**

- 折叠态只显示你最后保留勾选的那个维度
- 不是还显示旧维度
- 重启应用后，这个勾选状态还在

### 6. 验证拖拽和吸附

1. 用鼠标按住浮窗拖动。
2. 拖到屏幕最右边。
3. 看它是不是自动缩进去，只露出一个小把手。

**通过标准：**

- 能拖动
- 到边缘会自动吸附
- 不是卡住不动

### 7. 验证点击边缘把手恢复（这是必须人工补点的一项）

1. 浮窗吸附后，用鼠标点击露出来的小把手。

**通过标准：**

- 浮窗会重新弹出来
- 不会没反应
- 不会弹出来后位置错乱

### 8. 验证百炼 Mock 数据

1. 再去“设置”里添加“阿里云百炼”。
2. 随便填一个 Cookie 占位值保存。

**通过标准：**

- 浮窗里能出现百炼
- 展开后能看到 3 个维度
- 百分比和重置时间有内容

### 9. 验证错误状态

1. 把智谱 Token 故意改错。
2. 手动刷新一次，或者等自动刷新。

**通过标准：**

- 浮窗会显示错误提示
- 应用不会闪退
- 改回正确 Token 后能恢复正常

### 10. 验证重启恢复

1. 调整一下浮窗位置。
2. 改一下刷新频率。
3. 关闭应用，再重新启动。

**通过标准：**

- 浮窗位置还在
- 刷新频率还在
- 厂商配置还在
- 勾选状态还在

## 五、给最终结论时怎么判定

### 可以判定“通过”

满足下面 3 条就可以：

1. 自动化报告全部通过
2. 人工补点的 2 项（勾选维度、点击把手恢复）都通过
3. 真实智谱 Token 链路能正常显示线上额度

### 需要判定“未通过”

出现下面任意一种就算没过：

- 点击 checkbox 没反应
- 点击边缘把手没反应
- 重启后配置丢失
- 错误 Token 会把应用搞崩
- 浮窗不显示真实智谱数据
