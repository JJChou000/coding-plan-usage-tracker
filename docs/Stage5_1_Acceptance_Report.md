## 本次遇到的棘手问题

### 1. 已保存的窗口坐标可能落在当前屏幕之外

- 现象：更换显示器布局或分辨率后，应用启动时可能出现在可视区域外，表现为“像没启动”
- 解决方案：在 `src/main/window.ts` 启动窗口时增加坐标校正与可视区域约束，确保浮窗总能被拉回当前屏幕工作区
- 最终效果：窗口即使带着旧坐标启动，也会自动回到用户当前屏幕可见区域

### 2. 新增厂商后首次成功拉取数据时，折叠态维度可能为空

- 现象：配置刚保存、但 `checkedDimensions` 还是空数组时，折叠态可能拿不到主维度
- 解决方案：在 `src/renderer/context/AppContext.tsx` 首次收到有效维度数据时，自动补种第一个维度并同步持久化配置
- 最终效果：首次成功拉取后会自动选择一个可显示维度，折叠态不再空白

### 3. 展开态勾选控件与拖拽/点击折叠交互冲突

- 现象：早期实现里，勾选维度时偶发触发展开/折叠或拖拽逻辑，操作手感不稳定
- 解决方案：在 `src/renderer/components/ExpandedView.tsx` 将勾选控件改为显式 `button[role=\"checkbox\"]`，并补充事件阻断，避免与拖拽和折叠切换互相干扰
- 最终效果：维度勾选稳定，且最后一个已勾选维度仍会被保护，不会被误取消

### 4. 端对端验收需要稳定复现托盘与持久化流程

- 现象：仅靠手动点击很难稳定回归窗口恢复、托盘打开、重启后状态恢复等长链路
- 解决方案：补充 `scripts/acceptance/stage5_1_e2e.mjs`，并在主进程增加仅用于 E2E 的调试桥与独立 userData 目录支持
- 最终效果：阶段 5.1 的核心链路现在可以重复执行，后续回归时也能更快发现退化
