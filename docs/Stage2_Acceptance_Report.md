## 2. 验收前发现并已修复的问题

### 问题 1：正式 Electron 主窗口误用了 Stage 2 预览壳

验收时发现，真正启动的 Electron 浮窗不是直接渲染产品浮窗，而是把 `FloatingWindow` 包在了 “Stage 2.5” 预览页面里。

这会带来两个问题：

- 真机窗口里会出现不属于产品界面的阶段说明文字。
- 预览壳本身比浮窗内容更高，导致实际窗口内容被截断，看起来像“浮窗没显示完整”。

已修复：

- `src/renderer/App.tsx`
  - 预览模式下继续保留阶段预览页，方便做组件验收。
  - 真正的 Electron 运行模式下，主窗口直接渲染 `FloatingWindow`，设置窗口直接渲染 `SettingsPanel`。
  - 只在预览模式下注入 `PREVIEW_STATE`，避免正式窗口继续混用假数据。

### 问题 2：窗口最小尺寸限制会卡死折叠态和边缘把手

验收时继续对照 `2.5 FloatingWindow` 的要求检查，发现主进程里 `resizeWindow()` 的最小尺寸是：

- `MIN_WINDOW_WIDTH = 160`
- `MIN_WINDOW_HEIGHT = 80`

这会直接挡住两个阶段 2 要求：

- 折叠态高度不能真正收缩到组件计算值。
- 边缘吸附后的把手宽度是 `24px`，但窗口最小宽度被锁死在 `160px`，真实 Electron 窗口无法变成只露把手的状态。

已修复：

- `src/main/window.ts`
  - 将最小宽高改为 `HANDLE_WIDTH`，让折叠态和吸附把手尺寸可以真正生效。

### 问题 3：Hash 路由在预览环境下不会跟着变化

原实现里 `App.tsx` 只在渲染时直接读 `window.location.hash`，不会监听 `hashchange`。

这会影响两个验收动作：

- 从主预览页切换到 `#settings` 时，页面不会重新渲染。
- 设置页预览和主预览之间来回切换不稳定。

已修复：

- `src/renderer/App.tsx`
  - 新增 hash 路由状态同步逻辑，预览环境下切换 `#settings` 现在会即时刷新页面。

## 7. 人工验收怎么操作（小白版）

下面这部分你可以照着一步一步点，不需要懂代码。

### 第一步：启动项目

在项目根目录打开终端，运行：

```bash
npm run dev
```

你会看到：

- 桌面上出现一个小浮窗
- 右下角系统托盘出现应用图标

如果这两样都没有出现，就不算通过。

### 第二步：验收浮窗本体

请按下面顺序试：

1. 用鼠标拖动浮窗
2. 看它能不能跟着你移动
3. 点击浮窗内容区域
4. 看它能不能从“折叠态”切到“展开态”
5. 再点一次，看它能不能收回去
6. 把浮窗拖到屏幕最左边或最右边
7. 看它是不是会自动缩进去，只露出一个小把手
8. 点击那个小把手
9. 看它是不是会弹回来

通过标准（通俗版）：

- 能拖得动
- 点一下能展开，再点一下能收起
- 拖到边上会缩进去
- 点把手能恢复

### 第三步：验收设置面板

1. 找到右下角系统托盘里的应用图标
2. 右键它
3. 点“设置”
4. 看有没有打开一个设置窗口

通过标准：

- 能打开设置窗口
- 里面能看到“添加厂商”和“刷新频率”

### 第四步：验收添加/编辑/删除厂商

1. 在设置窗口点“添加厂商”
2. 选择一个厂商
3. 按要求填认证信息
4. 点“保存”
5. 看厂商卡片有没有出现在列表里
6. 再点“编辑”，改一下内容后保存
7. 再点“删除”
8. 看它是不是能从列表里消失

通过标准：

- 能新增
- 能编辑
- 能删除
- 列表变化是立刻可见的

### 第五步：验收刷新频率

1. 在设置窗口底部找到“刷新频率”
2. 把它从 `60s` 改成 `120s` 或其他选项
3. 保存后观察界面状态有没有跟着变

通过标准：

- 下拉框能选
- 选完后界面马上显示新的数值

### 第六步：验收“保存后浮窗立即反映变更”

最简单的看法是：

1. 打开设置
2. 添加一个厂商
3. 保存
4. 看浮窗里有没有马上出现对应厂商
5. 删除后再看浮窗有没有马上消失

通过标准：

- 不是“关掉重开才看到”
- 而是保存后，浮窗就能马上变
